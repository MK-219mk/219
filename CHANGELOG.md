# CHANGELOG — 219 Flashcard

All notable changes to this project will be documented in this file.

## [3.0.0] — 2025-07-10 (هنوز منتشر نشده)

> نکته: نسخه‌ی زنده‌ی فعلی سایت `2.1.0` است. این نسخه هنوز آپلود نشده.

### ✅ Fixed
- **حلقه‌ی بی‌پایان در مطالعه‌ی زودتر از موعد:** کارت‌هایی که فاصله‌ی کوتاه داشتند (مثلاً امتیاز «دوباره») بلافاصله دوباره در صف زودتر از موعد ظاهر می‌شدند؛ الان کارت‌هایی که همان روز مرور شده‌اند از این صف حذف می‌شوند
- **قفل‌شدن سهمیه‌ی روزانه:** مرورهای «زودتر از موعد» اشتباهاً در سهمیه‌ی روزانه‌ی کارت‌های due حساب می‌شدند و باعث می‌شدند کارت‌های واقعاً due آن روز مسدود شوند و پیام غلط «همه کارت‌ها مرور شده‌اند» نمایش داده شود؛ الان این مرورها جدا حساب می‌شوند
- **حذف حالت تمرین:** به‌خاطر پیچیدگی و رفتار غیرقابل‌پیش‌بینی، حالت تمرین از بخش مطالعه به‌طور کامل حذف شد (حالت مطالعه‌ی ساده در بخش «همه» جایگزین کافی است)
- **حذف کادر تبلیغاتی FSRS:** کادر بالای صفحه‌ی «همه کارت‌ها» حذف شد
- **ترجمه‌ی جمله‌ی خودکار نمایش داده نمی‌شد:** وقتی جمله‌ی مثال به‌صورت خودکار ساخته می‌شد، ترجمه‌اش در پشت کارت نمایش داده نمی‌شد (چون فقط جلوی کارت patch می‌شد نه کل کارت)؛ حل شد
- **ساخت جمله مثال دستی شد:** به‌جای ساخت خودکار (که باعث تأخیر و باگ نمایش می‌شد)، حالا یک دکمه روی کارت است که با کلیک، جمله ساخته می‌شود
- **علامت اشتباه حالت آفلاین:** ایموجی 📵 (که معنی «موبایل ممنوع» می‌دهد، نه قطع اینترنت) در همه‌جای سایت با 🔌 جایگزین شد

- **کلیک روی دسته:** حالا کل کارت دسته (نه فقط هدرش) قابل کلیک است و دسته را باز می‌کند
- **جهت صفحه بعد از رفرش:** وقتی زبان انگلیسی ذخیره شده بود، بعد از رفرش جهت صفحه اشتباهاً RTL می‌ماند؛ الان مستقیماً از تنظیمات اعمال می‌شود
- **زبان ناقص بعد از سوییچ:** toggleLang حالا صفحات cats/settings/stats و برچسب دسته‌ی انتخاب‌شده را هم به‌روزرسانی می‌کند
- **اتصال هوش مصنوعی:** کلید API حالا هنگام ذخیره و استفاده trim می‌شود (فاصله/خط جدید اضافه در کپی-پیست باعث خرابی URL و بروز خطای گمراه‌کننده‌ی «شبکه/CORS» می‌شد)؛ خطای HTTP 402 (اعتبار ناکافی) و 404 حالا با پیام مشخص نمایش داده می‌شوند و بدنه‌ی خطا در کنسول لاگ می‌شود
- **انیمیشن ساعت فیلیپ:** نیمه‌ی پایین ثابت هنگام تغییر عدد اشتباهاً بلافاصله عدد جدید را نشان می‌داد (در حالی که نیمه‌ی بالا هنوز قدیم بود) → ترکیب عدد خراب؛ قطعه‌ی دوم انیمیشن اشتباهاً روی نیمه‌ی بالا بود به‌جای پایین؛ اندازه‌ی فونت در حالت تمام‌صفحه/موبایل برای قطعات متحرک اعمال نمی‌شد. همه اصلاح شدند و انیمیشن حالا دو مرحله‌ی متوالی (نه هم‌پوشان) با پرسپکتیو درست دارد
- **FSRS Fields Preservation:** فیلد `created` هم در import حفظ می‌شود تا تاریخ اصلی ساخت کارت از بین نرود
- **Duplicate Detection:** الگوریتم تشخیص تکراری بدون حساسیت به بزرگی/کوچکی حروف و فاصله‌ی اضافه کار می‌کند
- **Badge Updates:** هر دو نشانگر `dot-study` و `dot-browse` درست به‌روزرسانی می‌شوند
- **Font Loading:** بارگذاری فونت Vazirmatn بهینه شد (`display=block` + dns-prefetch)
- **Card ID Collision:** تولید شناسه‌ی یکتا با شمارنده تقویت شد تا در ساخت سریع چند کارت تداخل نداشته باشد
- **Audio Toggle:** یکسان‌سازی تنظیم صدا (حذف تنظیم اضافه‌ی گیج‌کننده‌ی `featAudio`)
- **ورودی فایل (Import):** فایل‌های CSV و Anki/TSV حالا ستون‌های «جمله مثال»، «ترجمه جمله» و «توضیح» را هم تشخیص می‌دهند (قبلاً فقط کلمه+معنی+توضیح خوانده می‌شد و این دو ستون کاملاً نادیده گرفته می‌شدند)؛ اسم ستون‌ها هم به فارسی و هم انگلیسی پشتیبانی می‌شود

### 🚀 Added
- **سه سرویس هوش مصنوعی جدید:** Claude (Anthropic)، OpenRouter (دسترسی به ده‌ها مدل با یک کلید)، و OpenAI (ChatGPT) به زنجیره‌ی fallback اضافه شدند. ترتیب تلاش: DeepSeek → Gemini → Claude → OpenRouter → OpenAI → الگوی آفلاین. توجه: اتصال مستقیم OpenAI از مرورگر رسماً تضمین‌شده نیست و ممکن است گاهی با خطا مواجه شود (به همین دلیل آخرین گزینه قبل از حالت آفلاین است)
- **مطالعه‌ی زودتر از موعد (الگوریتم جدید):** وقتی کارت‌های امروز یک دسته تمام شد، دکمه‌ی «کارت‌های امروز تمام شد — مطالعه زودتر از موعد» جایگزین می‌شود و کارت‌ها را از نزدیک‌ترین موعد مرور تا دورترین می‌آورد (شبیه Learn Ahead در Anki). دکمه‌ی قدیمی «مرور پیش از موعد» که رفتار گیج‌کننده داشت حذف شد
- Package-lock.json برای build تکرارپذیر
- GitHub Actions CI/CD workflow
- فایل LICENSE (MIT)
- راهنمای کامل ساخت فایل ورودی (`219_import_guide.md`) + قالب خام CSV

### 🔒 Security
- کلید API قبل از ذخیره/استفاده trim می‌شود تا از خرابی URL جلوگیری شود
- اعتبارسنجی بهتر URL در Electron
- تشخیص تکراری قوی‌تر
